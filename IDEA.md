# 플레이스 SEO 최적화 자동화 툴

## 한 줄 정의
네이버 플레이스 URL만 넣으면, 섹션별 최적화 프롬프트와 원본 정보가 담긴 "작업지"를 생성하여, 사람이 Antigravity/Claude에 붙여넣어 결과를 받고, 그 결과를 플레이스에 다시 붙여넣는 것만으로 SEO 최적화가 완성되는 반자동 툴.

## 해결하는 문제
- 네이버 플레이스 SEO는 섹션(소개글/키워드/메뉴설명/답글톤 등)마다 다른 프롬프트와 노하우가 필요함
- 매번 플레이스 정보를 직접 복사해서 LLM에 넣고, 섹션별로 프롬프트를 바꿔가며 작업하는 것이 번거로움
- 여러 섹션을 관리하다 보면 구조가 복잡해지고 프롬프트 재사용이 어려움

## 해결 방식
1. 네이버 플레이스 URL을 입력받아 현재 정보를 자동 추출
2. 섹션별 프롬프트 템플릿(YAML)을 읽어서
3. 추출된 정보를 프롬프트에 끼워넣은 "작업지(markdown)" 파일을 생성
4. 사용자는 이 작업지의 각 섹션 프롬프트를 복사 → LLM에 붙여넣기 → 결과를 받음
5. 결과를 플레이스에 수동으로 붙여넣어 적용

## 범위 (In Scope)
- 네이버 플레이스 읽기 전용 스크래핑 (업체명/카테고리/주소/영업시간/소개글/해시태그 등)
- 섹션별 YAML 프롬프트 템플릿 관리
- 사용자가 직접 편집 가능한 섹션 추가/수정
- 작업지(markdown) 파일 생성
- 점진적 섹션 확장

## 비범위 (Out of Scope) — 하지 않음
- 자동 로그인, 자동 등록, 자동 포스팅 (네이버 ToS 위반, 법적 리스크)
- 자동 리뷰 작성/답글 게시 (사람이 확인 후 수동 적용)
- 계정 관리, 봇 운영
- 여러 플레이스 일괄 처리 (초기 단계에서는 1개씩)
- LLM API 직접 호출 (Phase 1에서는 Antigravity/Claude에 사람이 붙여넣어 처리)

## 기술 스택 (초기)
- 언어: Python 3.11+
- 의존성 관리: uv
- 스크래핑: Playwright (Chromium)
- 프롬프트 템플릿: YAML
- 출력 형식: Markdown
- LLM: Phase 1은 외부 (Antigravity/Claude 채팅창). Phase 2에서 API 직접 호출 고려

## 단계별 로드맵
- **Phase 0 (현재)**: 1개 URL → 스크래핑 → 2~3개 섹션 작업지 생성 → 터미널 실행
- **Phase 1**: 섹션 추가 (메뉴/답글/이벤트 등), 스크래핑 정확도 개선
- **Phase 2**: 간단한 UI (Streamlit) 또는 LLM API 연동으로 완전 자동화
- **Phase 3**: 여러 플레이스 일괄, 결과 저장/비교, 팀 공유

## 핵심 원칙
- 한 번에 한 가지씩 추가 (작은 변경, 자주 실행)
- 동작하는 것 > 완벽한 구조
- 프롬프트는 사용자 SEO 노하우가 들어가는 자리 — 코드는 그 그릇일 뿐
- 자동화 봇이 되지 않기 (읽기 전용 + 사람의 복붙)
